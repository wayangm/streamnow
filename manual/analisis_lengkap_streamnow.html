Analisis Lengkap Proyek StreamNow

1. Tujuan Utama Proyek StreamNow

StreamNow adalah aplikasi live streaming berbasis web yang memungkinkan pengguna melakukan live streaming ke berbagai platform seperti YouTube, Facebook, Twitch, TikTok, Shopee Live, dan platform lainnya menggunakan protokol RTMP. Aplikasi ini dirancang untuk dijalankan di VPS (Virtual Private Server) dan menyediakan fitur-fitur untuk mengelola video dan streaming secara efektif.

2. Struktur Arsitektur Aplikasi

StreamNow menggunakan arsitektur berbasis Node.js dengan pola MVC (Model-View-Controller). Struktur direktori utama terdiri dari:

  • app.js: File utama aplikasi yang berisi konfigurasi Express, rute, middleware, dan inisialisasi
  • models/: Berisi model-model data (User, Video, Stream, Playlist)
  • views/: Template EJS untuk tampilan (dashboard, gallery, settings, dll)
  • services/: Layanan utama (streaming, logging, monitoring, scheduler)
  • middleware/: Middleware kustom untuk upload file
  • utils/: Fungsi utilitas (storage, video processing, Google Drive integration)
  • db/: File database SQLite dan konfigurasi database
  • public/: File statis (CSS, JavaScript, gambar)
  • package.json: Dependensi dan skrip proyek

Aplikasi ini menggunakan SQLite sebagai database, EJS sebagai templating engine, dan Tailwind CSS untuk styling.

3. Teknologi dan Dependensi yang Digunakan

Proyek StreamNow menggunakan berbagai teknologi dan dependensi penting:

Backend Framework & Tools:

  • Node.js (v22 atau versi terbaru)
  • Express.js sebagai framework web
  • SQLite3 sebagai database
  • EJS sebagai templating engine
  • EJS-Mate untuk layout template

Video Processing:

  • FFmpeg (melalui @ffmpeg-installer/ffmpeg dan fluent-ffmpeg)
  • get-video-duration untuk mendapatkan informasi durasi video
  • file-type untuk identifikasi tipe file

Autentikasi & Security:

  • bcrypt untuk hashing password
  • CSRF protection
  • express-session untuk manajemen sesi
  • connect-sqlite3 untuk menyimpan sesi di SQLite

File Upload & Transfer:

  • Multer untuk upload file
  • @tus/server dan @tus/file-store untuk upload chunked (resumable upload)
  • axios untuk HTTP client
  • fs-extra untuk operasi file system

UI & Styling:

  • Tailwind CSS (melalui CDN)
  • Video.js untuk player video
  • Tabler Icons untuk icon
  • Google Fonts (Inter)

Monitoring & Sistem:

  • systeminformation (si) untuk informasi sistem
  • os-utils untuk informasi OS
  • express-rate-limit untuk rate limiting

Utilities:

  • dotenv untuk manajemen environment variable
  • uuid untuk generate ID unik
  • express-validator untuk validasi input

4. Analisis Fitur-fitur Utama Aplikasi

StreamNow memiliki berbagai fitur komprehensif untuk kebutuhan live streaming:

Fitur Manajemen Pengguna:

  • Registrasi dan login pengguna
  • Sistem otentikasi berbasis sesi
  • Role-based access (admin/member)
  • Profile management dengan avatar
  • Ganti password dan pengaturan akun
  • Manajemen pengguna (hanya untuk admin)

Fitur Live Streaming:

  • Multi-platform streaming (YouTube, Facebook, Twitch, TikTok, Shopee Live, dll)
  • Koneksi ke RTMP server
  • Live streaming dari file video yang di-upload
  • Live streaming dari playlist video
  • Penjadwalan streaming
  • Mode lanjutan (advanced mode) dengan pengaturan bitrate, resolusi, dan FPS

Fitur Upload & Manajemen Video:

  • Upload video file ke server
  • Import video dari Google Drive
  • Generate thumbnail otomatis
  • Metadata extraction (durasi, resolusi, bitrate, fps)
  • Preview video sebelum streaming
  • Manajemen file video (rename, delete)

Fitur Playlist:

  • Pembuatan playlist dari video yang tersedia
  • Mode acak (shuffle) dan urutan
  • Manajmen urutan video dalam playlist
  • Preview playlist

Fitur Monitoring & Analisis:

  • Monitoring sistem (CPU, RAM, disk usage, network)
  • Dashboard real-time
  • Riwayat streaming
  • Statistik kinerja

Fitur Penjadwalan & Otomatisasi:

  • Penjadwalan waktu streaming
  • Auto-stop berdasarkan durasi
  • Scheduler service untuk menjalankan stream terjadwal

5. Sistem Autentikasi dan Manajemen Pengguna

Sistem autentikasi StreamNow menggunakan pendekatan berbasis sesi dengan SQLite sebagai penyimpanan sesi:

Proses Autentikasi:

  • Menggunakan express-session dengan SQLite sebagai penyimpanan
  • Password di-hash menggunakan bcrypt
  • Sistem role-based access (admin/member)
  • Status akun (aktif/tidak aktif)

Fitur Manajemen Pengguna:

  • Registrasi akun baru dengan avatar
  • Login dengan validasi CSRF
  • Setup akun pertama kali (untuk admin pertama)
  • Reset password (melalui script reset-password.js)
  • Manajemen profil (username, avatar, password)
  • Admin dapat mengelola akun pengguna lain (ubah role, status, delete)

Keamanan Autentikasi:

  • Rate limiting untuk login (maksimal 5 kali percobaan dalam 15 menit)
  • Delay 1 detik setelah setiap login attempt
  • Validasi input yang ketat
  • CSRF protection untuk formulir
  • Session timeout 24 jam

6. Sistem Streaming Video

Sistem streaming dalam StreamNow dibangun untuk mendukung pengiriman video langsung ke berbagai platform RTMP:

Proses Streaming:

  • Menggunakan FFmpeg untuk encoding dan transmisi video
  • Mode sederhana (passthrough) dan mode lanjutan (re-encoding)
  • Mendukung berbagai resolusi (720p, 1080p, 1440p, 4K)
  • Pengaturan bitrate (2500kbps hingga 20000kbps)
  • Pengaturan FPS (30, 60, 120)
  • Dukungan orientasi (landscape/portrait)

Fitur Streaming:

  • Loop video selama streaming
  • Preview video sebelum streaming
  • Playlist support (stream video secara berurutan atau acak)
  • Pengaturan platform dengan template RTMP
  • Monitoring status streaming (live/scheduled/offline)

Implementasi Teknis:

  • Menggunakan child_process.spawn untuk menjalankan FFmpeg
  • Sistem retry otomatis jika FFmpeg crash
  • Manajemen proses FFmpeg dengan penanganan error
  • Logging detail selama proses streaming
  • Penyimpanan riwayat streaming ke database

7. Sistem Manajemen File dan Video

Sistem manajemen file dan video dalam StreamNow mencakup berbagai fitur untuk penanganan file media:

Upload File:

  • Upload video file menggunakan Multer
  • Tipe file yang didukung: MP4, AVI, MOV
  • Penamaan unik untuk file yang di-upload
  • Validasi ukuran file (maksimal 10GB)

Manajemen Video:

  • Generate thumbnail otomatis dari video
  • Ekstraksi metadata (durasi, resolusi, bitrate, fps)
  • Penyimpanan lokasi file dalam database
  • Dukungan import dari Google Drive

Struktur Direktori:

  • public/uploads/videos/ - untuk file video
  • public/uploads/thumbnails/ - untuk thumbnail
  • public/uploads/avatars/ - untuk avatar pengguna

Fitur File:

  • Penyimpanan file secara lokal di server
  • Penghapusan otomatis file ketika video dihapus
  • Dukungan untuk playlist video
  • Pratinjau video sebelum streaming

8. Sistem Penjadwalan dan Manajemen Stream

Sistem penjadwalan dan manajemen stream merupakan fitur penting dalam StreamNow:

Fitur Penjadwalan:

  • Penjadwalan stream berdasarkan tanggal dan waktu
  • Pengaturan durasi streaming
  • Auto-start stream sesuai jadwal
  • Auto-stop stream setelah durasi berakhir

Implementasi Scheduler:

  • Menggunakan schedulerService.js untuk manajemen penjadwalan
  • Cek berkala setiap 60 detik untuk stream terjadwal
  • Penggunaan setTimeout untuk auto-stop berdasarkan durasi
  • Penanganan stream yang melebihi durasi yang ditentukan

Manajemen Stream:

  • Status stream (live, scheduled, offline)
  • Manajemen daftar stream dalam antarmuka
  • Fungsi start/stop/cancel untuk berbagai status
  • Riwayat streaming yang tersimpan dalam database

Fitur Lain:

  • Penjadwalan ulang stream jika gagal
  • Manajemen stream aktif dan proses FFmpeg
  • Penyimpanan metadata stream (platform, pengaturan, dll)

9. Sistem Database dan Model Data

StreamNow menggunakan SQLite sebagai database dengan struktur tabel sebagai berikut:

Tabel-tabel Utama:

  • users: Untuk menyimpan informasi pengguna (username, password hash, role, status, avatar, dll)
  • videos: Untuk menyimpan informasi video (judul, path, thumbnail, metadata, dll)
  • streams: Untuk menyimpan informasi stream (judul, RTMP URL, stream key, platform, status, dll)
  • stream_history: Untuk menyimpan riwayat streaming
  • playlists: Untuk menyimpan informasi playlist
  • playlist_videos: Tabel junction untuk hubungan banyak-ke-banyak antara playlist dan video
  • sessions: Tabel untuk menyimpan sesi pengguna (melalui connect-sqlite3)

Model Data:

  • User.js: Manajmen akun pengguna, termasuk registrasi, login, dan otorisasi
  • Video.js: Manajemen file video, termasuk upload dan metadata
  • Stream.js: Manajemen stream, termasuk status dan penjadwalan
  • Playlist.js: Manajemen playlist video

Fitur Database:

  • Relasi antar tabel (videos ke users, streams ke videos/users, dll)
  • Indeks untuk performa pencarian
  • Validasi konsistensi data

10. Sistem Logging dan Monitoring

Sistem logging dan monitoring dalam StreamNow mencakup dua aspek penting:

Sistem Logging:

  • File logger.js yang mencatat semua output console ke file log
  • Penyimpanan log di folder `logs/app.log`
  • Penanganan berbagai level log (info, warn, error, debug)
  • Logging unhandled rejection dan uncaught exception

Sistem Monitoring:

  • Penggunaan library `systeminformation` (si) untuk mendapatkan data sistem
  • Monitoring CPU, RAM, disk usage, dan network
  • Tampilan real-time di dashboard admin
  • Penggunaan fungsi `getSystemStats()` di services/systemMonitor.js
  • Tampilan data sistem dalam bentuk grafik dan indikator

Fitur Monitoring:

  • Pemantauan sumber daya server secara real-time
  • Penghitungan kecepatan jaringan (upload/download)
  • Informasi kapasitas dan penggunaan disk
  • Tampilan dalam tampilan dashboard

12. Cara Instalasi dan Deployment

StreamNow menyediakan beberapa metode instalasi dan deployment:

Instalasi Otomatis:

  • Script `install.sh` untuk instalasi otomatis di server
  • Mengupdate sistem, menginstal Node.js, FFmpeg, Git
  • Mengkloning repositori dan menginstal dependensi
  • Membuat secret untuk session
  • Mengkonfigurasi firewall dan PM2

Instalasi Manual:

  • Instalasi Node.js (versi 20 atau lebih baru)
  • Instalasi FFmpeg untuk pemrosesan video
  • Instalasi Git
  • Clone repositori: `git clone https://github.com/bangtutorial/streamnow`
  • Install dependensi: `npm install`
  • Generate secret: `node generate-secret.js`
  • Jalankan aplikasi: `npm run dev` atau `pm2 start app.js`

Docker Deployment:

  • Disediakan Dockerfile dan docker-compose.yml
  • Build image dengan: `docker-compose up --build`
  • Port 7575 diakses melalui localhost:7575
  • Volume mounting untuk persistensi data

Konfigurasi:

  • File `.env` untuk konfigurasi environment (PORT, SESSION_SECRET)
  • Port default 7575 (dapat diubah di .env)
  • Konfigurasi timezone server (disarankan Asia/Jakarta)
  • Konfigurasi firewall untuk membuka port SSH dan aplikasi

13. Keamanan dan Best Practice

StreamNow menerapkan beberapa aspek keamanan dan best practice:

Keamanan Autentikasi:

  • Password di-hash menggunakan bcrypt dengan salt 10 putaran
  • CSRF protection untuk formulir penting
  • Rate limiting untuk login (maksimal 5 kali percobaan dalam 15 menit)
  • Session timeout 24 jam
  • Tidak menyimpan password dalam bentuk plain text

Keamanan Input:

  • Validasi input menggunakan express-validator
  • Sanitasi data sebelum disimpan ke database
  • Validasi tipe file dan ukuran pada upload

Keamanan Sistem:

  • Tidak menyimpan file .env di repository (ada di .gitignore)
  • Menggunakan secure session cookie dalam mode production
  • Validasi file yang di-upload
  • Penanganan error yang tidak bocor informasi sensitif

Best Practice:

  • Menggunakan UUID untuk ID unik
  • Penamaan file unik untuk mencegah konflik
  • Sistem logging untuk debugging
  • Error handling komprehensif
  • Menggunakan Promise dan async/await dengan benar
  • Struktur kode modular dengan MVC pattern

14. Ringkasan Keseluruhan Proyek

StreamNow adalah aplikasi live streaming yang komprehensif dan dirancang untuk dijalankan di VPS. Aplikasi ini menyediakan solusi end-to-end untuk streaming video ke berbagai platform RTMP seperti YouTube, Facebook, Twitch, dan lainnya.

Keunggulan Utama:

  1. Multi-platform Streaming: Mendukung streaming ke berbagai platform populer
  2. Manajemen Video Lengkap: Upload, import Google Drive, playlist, dan metadata
  3. Sistem Penjadwalan: Fitur penjadwalan dan auto-stop untuk streaming otomatis
  4. UI/UX Responsif: Tampilan modern dan responsif untuk berbagai perangkat
  5. Monitoring Sistem: Dashboard real-time untuk memantau sumber daya server
  6. Keamanan: Implementasi keamanan yang baik untuk otentikasi dan data

Arsitektur:

  • Backend: Node.js + Express.js
  • Database: SQLite
  • Frontend: EJS template + Tailwind CSS + JavaScript
  • Video Processing: FFmpeg

Fitur-fitur Kunci:

  • Live streaming ke berbagai platform RTMP
  • Upload dan manajemen video
  • Penjadwalan streaming
  • Playlist video
  • Monitoring sistem real-time
  • Multi-user dengan role-based access
  • Import dari Google Drive
  • Mode advanced dan simple untuk streaming

Aplikasi ini sangat cocok untuk pengguna yang membutuhkan solusi streaming yang handal dan dapat dijalankan di server pribadi, dengan tampilan yang modern dan fitur-fitur lengkap untuk manajemen streaming.
